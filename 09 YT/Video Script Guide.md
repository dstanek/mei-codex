# Video Script Guide

## Shorts (under 60 seconds, ideally under 30 seconds)

- 4–5 bullets max per topic outline — tight and focused on a single idea
- Open with a relatable pain point or hook, not an introduction
- One concept only — no tangents or "also worth mentioning"
- End with a clear payoff: a tip, a command, or a tease to the long-form video
- If the short overlaps with a long-form video, frame it explicitly as a teaser so both pieces earn their place

## Long-Form Videos

- Enough depth to fill a structured walkthrough — typically 6–10 topic bullets
- Each video should have a clear scope that doesn't heavily overlap with sibling videos
- Structure: problem → install/setup → core concept → demo → gotchas/tips
- End with a natural next step (related video, docs link, or follow-up concept)

## Note Structure

- Title: `# series-name: Video Title` — keeps notes organized when viewed outside their folder
- `## Outline` — bullet list written before scripting; defines scope (see Topic Bullet Lists below)
- `## Script` — written script goes here once the outline is solid

## Topic Bullet Lists (Pre-Script Stage)

- Write bullets before scripts — they define scope and catch overlap early
- Each bullet = one thing to say or show on screen, not a full paragraph
- If a bullet needs sub-bullets, it may be too big and should be split into its own section
- Review bullets for shorts against the time budget: ~3–5 seconds per bullet
